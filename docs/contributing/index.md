# Contributing

We welcome contributions to this project! Please read the following guidelines before contributing.

## How to Contribute

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Submit a pull request

## Guidelines

- Follow the existing code style
- Write clear commit messages
- Test your changes thoroughly
- Ensure all Markdown files have proper front matter
- Use folder-based navigation (link to folders, not `.md` files)
- Verify asset paths use `relative_url` filter

## Filing Issues

For bugs, feature requests, or documentation improvements, please open an issue in the repository.

## Community

We appreciate all contributions and value the input of our community members. Thank you for helping improve this resource!
